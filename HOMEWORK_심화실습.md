# 과제 1. 파트너지원 / CSP — 월 정산 대사 자동화

## 🎯 시나리오
파트너사 N곳의 결제 데이터와 NHN 내부 매출 데이터를 매월 맞춰보는(대사) 일을 합니다. 보통 엑셀 VLOOKUP으로 1시간 이상 걸리는데, 한 번에 자동화해 봅시다.

## 📥 입력
- `partner_revenue.xlsx` (파트너 결제, ~500행: 파트너ID, 정산월, 금액)
- `nhn_internal.xlsx` (NHN 내부 매출, ~500행: 동일 키)
- (없으면 강사 제공 샘플 사용)

## 📤 산출물
- `대사_차이_보고서.xlsx` (3개 시트)
  - Sheet 1: 매칭 + 금액 일치
  - Sheet 2: 매칭 + 금액 불일치 (차이 금액 컬럼)
  - Sheet 3: 한쪽만 존재
- `요약.md` — 두괄식, 총건수 / 차이 건수 / 차이 합계 / Top 5

## 🪜 단계별 프롬프트

**Step 1 — 데이터 구조 파악**
```
@partner_revenue.xlsx 와 @nhn_internal.xlsx 의 컬럼 구조와
데이터 분포를 보여줘. 파트너ID + 정산월이 대사 키로 적절한지 확인해줘.
```

**Step 2 — 대사 작업 (outer join)**
```
두 파일을 파트너ID + 정산월 기준으로 outer join 해서 대사해줘.
다음 3개 그룹으로 분류:
  1) 양쪽 존재 + 금액 일치
  2) 양쪽 존재 + 금액 불일치 (차이 컬럼 추가)
  3) 한쪽만 존재 (어느 쪽인지 표시)
결과는 대사_차이_보고서.xlsx의 3개 시트로 저장.
```

**Step 3 — 요약 보고**
```
방금 만든 대사 결과를 요약.md에 두괄식으로 정리해줘.
총건수, 차이 건수, 차이 합계, 가장 큰 차이 Top 5를 표로.
끝에 "다음 액션 3가지"를 결재자 관점에서 적어줘.
```

## ✅ 평가 기준
- ⭐ 차이 항목을 정확히 분리했는가
- ⭐⭐ 차이 사유(반올림/환율/누락)를 분류했는가
- ⭐⭐⭐ Dooray 위키에 자동 업로드까지 했는가

## 🚀 보너스 도전
- 매월 반복되는 차이 패턴이 있는 파트너 식별 ("이 파트너는 매월 1만원씩 차이남")
- `cron` 또는 Windows 작업 스케줄러로 매월 1일 자동 실행
- 차이 발생 시 담당자에게 Dooray 메신저 자동 알림

## 📖 활용 어휘
- **대사 (reconciliation)** — 두 장부를 맞춰보는 작업
- **outer join** — 양쪽 데이터를 모두 유지하면서 결합 (한쪽에만 있는 행도 포함)
- **두괄식 / BLUF** — 결론을 먼저 제시하는 글 구조

# 과제 2. 경영관리 / 총무 — 계약서 독소조항 검토 봇

## 🎯 시나리오
협력사·벤더와 새 계약을 맺기 전, 법무팀에 보내기 전 1차 검토를 합니다. 일방에 매우 불리한 조항(독소조항)을 빠르게 찾아내야 합니다.

## 📥 입력
- `계약서_초안.pdf` 또는 `.docx` (실제 진행 중인 계약 or 강사 제공 샘플)
- `사내_표준조항.md` (있으면 — 없으면 Claude가 일반 기준 사용)

## 📤 산출물
- `독소조항_검토_보고서.md`
  - 발견된 조항 목록 (조항 번호, 원문, 위험도 ★~★★★)
  - 각 조항별: **무엇이 문제인가** / **수정 제안** / **법무팀 확인 필요 여부**
- `법무팀_전달_요약.md` — 1페이지로 압축

## 🪜 단계별 프롬프트

**Step 1 — 갑/을 관계 식별**
```
@계약서_초안.pdf 에서 갑/을 관계를 먼저 식별해줘.
우리가 어느 쪽인지, 의무·책임이 어느 쪽에 더 쏠려있는지 한 줄로.
```

**Step 2 — 독소조항 체크**
```
이 계약서에서 다음 관점으로 독소조항을 체크해줘:
  1) 일방적 해지권 (상대방만 가능)
  2) 손해배상 무제한 또는 상한 부재
  3) 비밀유지 의무 비대칭
  4) 지식재산권 일방 귀속
  5) 부당하게 짧은 통지 기간
  6) 자동 갱신 + 해지 곤란 조항
각 조항에 위험도 ★~★★★를 매기고 수정 제안을 함께 써줘.
```

**Step 3 — 법무팀 전달용 압축**
```
방금 보고서를 1페이지 분량의 법무팀_전달_요약.md로 압축해줘.
역피라미드 구조로 — 결론(머지/조건부/거부) → 핵심 3건 → 상세 링크.
```

## ✅ 평가 기준
- ⭐ 독소조항 후보 3개 이상 발견
- ⭐⭐ 위험도 평가 + 수정 제안 포함
- ⭐⭐⭐ 사내 표준조항과 자동 대조까지

## 🚀 보너스 도전
- 사내 표준조항 라이브러리를 `CLAUDE.md`에 등록 → 매번 자동 참조
- 검토 완료된 계약은 Dooray 드라이브의 "계약/검토완료" 폴더로 자동 이동

## 📖 활용 어휘
- **독소조항 (poison clause)** — 일방에 매우 불리한 조항
- **갑을 관계** — 계약 당사자 간 권리·의무 비대칭 구조
- **역피라미드 구조** — 가장 중요한 정보부터 점차 디테일로 (뉴스 기사 방식)

# 과제 3. 인사 — BEI 면접 키트 자동 생성

## 🎯 시나리오
새 포지션 채용 시 직무기술서(JD)에 맞는 면접 질문을 만들어야 합니다. 단순 "강점이 뭐예요?" 같은 질문 말고, 과거 실제 행동을 묻는 **BEI(행동사건면접)** 방식으로요.

## 📥 입력
- `JD_백엔드시니어.md` (또는 본인 팀이 채용 중인 포지션)
- `핵심역량_사내정의.md` (있으면)

## 📤 산출물
- `면접_키트.md`
  - 핵심 역량별 BEI 질문 5개씩 × 4개 역량 = 20문항
  - 각 질문마다: 좋은 답변의 특징 / Red flag / 후속 질문(probing)
- `평가표.xlsx` — 면접관이 실시간으로 점수 매기는 양식

## 🪜 단계별 프롬프트

**Step 1 — JD에서 핵심 역량 추출**
```
@JD_백엔드시니어.md 에서 핵심 역량 4가지를 추출해줘.
역량마다 직무 맥락에서 무엇을 의미하는지 한 줄 정의도 함께.
```

**Step 2 — BEI 질문 생성**
```
방금 뽑은 4개 역량 각각에 대해 BEI 질문 5개씩 만들어줘.
형식: "최근 ~한 상황에서 [구체적 행동]을 한 경험을 말씀해주세요"
STAR 구조(Situation/Task/Action/Result)로 답이 나올 수 있게.
각 질문 아래에:
  - 좋은 답변의 특징 (구체적 행동 + 본인 역할 명확)
  - Red flag (가설형 답변, "할 것 같아요" 같은 추상 답)
  - Probing 후속 질문 2개
```

**Step 3 — 평가표 작성**
```
면접관이 실시간으로 채점하는 평가표.xlsx 만들어줘.
컬럼: 후보자명, 역량별 점수(1-5), 특이사항, 추천여부.
시트 2개: 1) 평가표 양식, 2) 채점 기준표.
```

## ✅ 평가 기준
- ⭐ 가설형이 아닌 행동 기반 질문 20개
- ⭐⭐ Red flag와 probing이 구체적
- ⭐⭐⭐ JD 외 사내 컬처핏 질문 5개 추가

## 🚀 보너스 도전
- 면접 종료 직후 음성 녹취 → 자동 회의록 → STAR 구조로 분류 → 평가표 자동 채점
- Dooray 위키에 채용 라운드별 결과 자동 정리

## 📖 활용 어휘
- **BEI 질문 (Behavioral Event Interview)** — 과거 실제 행동을 묻는 면접 기법
- **STAR 구조** — Situation/Task/Action/Result로 답을 구조화
- **컬처핏 (culture fit)** — 조직 문화와의 정렬도

# 과제 4. 마케팅 — 캠페인 카피 A/B 변형 + 트렌드 매칭

## 🎯 시나리오
새 캠페인 카피 1개를 받았는데, PAS 구조로 변형하고 채널별 톤에 맞춰 A/B 테스트를 하고 싶습니다. 그리고 최근 트렌드 키워드를 자동으로 매칭하고 싶어요.

## 📥 입력
- `원본_카피.md` (강사 제공 또는 본인 진행 카피)
- `채널별_톤가이드.md` (있으면)

## 📤 산출물
- `카피_변형본.md`
  - A안: 원본 톤 유지 + 카피 다듬기
  - B안: PAS 구조로 재작성 (Problem → Agitation → Solution)
  - 채널별(인스타·블로그·메일) × 2안 = **6개 변형**
- `예상_성과_분석.md` — 각 변형의 예상 CTR/전환 근거

## 🪜 단계별 프롬프트

**Step 1 — 원본 카피 진단**
```
@원본_카피.md 를 다음 관점으로 진단해줘:
  - 현재 사용 중인 카피 구조 (AIDA / PAS / FAB / 기타)
  - Hook 강도 (1-10)
  - 행동 유도(CTA) 명확성
  - AI 티 (humanize 필요 여부)
```

**Step 2 — A/B 변형 생성**
```
A안: 원본 톤 유지 + 어색한 부분만 다듬기
B안: PAS 구조로 완전 재작성
  - Problem: 타겟의 실제 고민 한 줄
  - Agitation: 그 고민이 방치되면 어떻게 되는지
  - Solution: 우리 제품이 그것을 어떻게 해결하는지
각각을 인스타(180자) / 블로그 도입부(300자) / 메일 제목+본문 첫 줄
3개 채널에 맞게 변형해서 6개 버전 만들어줘.
```

**Step 3 — 트렌드 매칭 (선택)**
```
지금 마케팅 업계에서 떠오르는 키워드 중 우리 카피와 자연스럽게
연결할 수 있는 것 3개 제안해줘. 각 키워드별로:
  - 매칭 이유
  - 어떤 변형 버전에 어떻게 끼워넣을지
  - 주의할 점 (남용 시 역효과)
```

## ✅ 평가 기준
- ⭐ A/B 각 1개씩 + 채널 변형 6개 모두 생성
- ⭐⭐ 각 변형의 예상 성과 근거 제시
- ⭐⭐⭐ 트렌드 매칭 + AI 티 제거 통과 (humanize)

## 🚀 보너스 도전
- 6개 변형을 실제 인스타/메일에 발행 후 1주일 뒤 결과를 Claude에게 다시 분석시키기
- 가장 잘 된 변형의 패턴을 `marketing_playbook.md`에 적립

## 📖 활용 어휘
- **PAS 구조** — Problem → Agitation → Solution
- **AIDA** — Attention → Interest → Desire → Action
- **Hook** — 첫 3초~한 줄 안에 주의를 끄는 장치
- **humanize** — AI 티(번역투·기계적 병렬)를 사람이 쓴 글처럼 다듬는 작업

# 과제 5. 보안 / IT지원 — 접근권한 감사 + 이상 행동 탐지

## 🎯 시나리오
사내 시스템 접근 로그가 매일 수십만 건 쌓입니다. 분기에 한 번 권한 감사를 하고, 이상 행동(off-hours 접근, 비정상 패턴)을 찾아야 합니다.

## 📥 입력
- `access_log_sample.csv` (강사 제공 익명 샘플, ~10000행)
- 컬럼: timestamp, user_id, resource, action, source_ip
- `사용자_부서_매핑.xlsx` (선택)

## 📤 산출물
- `권한감사_보고서.md`
  - 부서별 접근 통계 (정상 vs 이상치)
  - 이상 행동 Top 10 (사유 + 위험도)
  - 권한 조정 권고 사항
- `dashboard.html` — 한 페이지로 보는 시각화 대시보드

## 🪜 단계별 프롬프트

**Step 1 — EDA (탐색적 분석)**
```
@access_log_sample.csv 의 구조를 파악하고 EDA 해줘:
  - 시간대별 / 부서별 / 액션별 분포
  - 결측치 / 중복 / 비정상 값 체크
  - 분포의 왜도(skewness)도 확인
```

**Step 2 — 이상치 탐지**
```
다음 관점으로 이상치 탐지해줘:
  1) Off-hours 접근 (22:00 ~ 06:00) — 부서별 평균과 비교
  2) IP 이상 (사내 IP 대역 외부)
  3) 짧은 시간 내 대량 접근 (rate anomaly)
  4) 평소 안 쓰던 리소스 접근 (behavioral drift)
각각을 IQR 또는 Z-score 기반으로 탐지하고,
탐지된 행만 모은 anomalies.csv 만들어줘.
```

**Step 3 — 보고서 + 대시보드**
```
방금 결과를 권한감사_보고서.md로 정리해줘. 두괄식:
  결론(전반 상태) → 이상 행동 Top 10 → 권고사항 3가지.

별도로 dashboard.html을 만들어줘 — 시간대별 히트맵, 부서별 박스플롯,
이상치 산점도. Chart.js나 plotly 사용, 한 페이지에 다 들어가게.
```

## ✅ 평가 기준
- ⭐ 4가지 이상 탐지 관점 중 2개 이상 적용
- ⭐⭐ 이상치 + 사유 명시
- ⭐⭐⭐ 대시보드까지 + 자동 갱신 스크립트화

## 🚀 보너스 도전
- 발견된 이상 행동의 사용자를 Dooray로 자동 알림 (보안팀 채널)
- 다음 분기 감사 시 변화 추적을 위한 baseline 저장 → 매분기 자동 diff
- 가짜 양성(false positive) 패턴을 학습시켜 점차 정확도 올리기

## 📖 활용 어휘
- **EDA (Exploratory Data Analysis)** — 본격 분석 전 데이터 구조·분포 파악
- **이상치 (outlier)** — 통계적으로 평균에서 벗어난 값
- **IQR (Interquartile Range)** — 사분위수 범위, 이상치 탐지 기준
- **Z-score** — 평균에서 표준편차 몇 배 떨어졌는지
- **behavioral drift** — 평소 패턴에서 벗어난 행동 변화
