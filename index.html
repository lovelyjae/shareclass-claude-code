기본 설치 (계정 + Node.js + Claude Code)

할 것: Claude 계정 확인 → Node.js 설치 → Claude Code 설치. 이 3개만 하면 실습 준비 끝!
목표: 터미널에서 claude를 실행했을 때 "Welcome to Claude Code!"가 뜨는 것!
교육 당일 같이 설치해도 괜찮습니다! 미리 해오시면 실습에 더 집중할 수 있어요. 막히면 두레이로 편하게 연락주세요!

1단계: Claude 유료 계정 확인

교육에 참여하려면 Claude 유료 계정(Pro/Team/Max)이 필요합니다.

확인 방법:

  • 브라우저에서 claude.ai 접속
  • 로그인
  • 좌측 하단에 "Pro", "Team", 또는 "Max"가 보이면 OK
좌측 하단에 플랜 표시가 안 보이는 경우: 좌측 하단 프로필 아이콘 → Settings → Billing 에서 현재 플랜을 확인할 수 있습니다.

아직 유료 계정이 없으신 분은 5/15(금)부터 약 1개월 간, Team(25$) 요금제 계정이 제공될 예정입니다. 지원 시, 추가 개별 안내드릴 예정이니 걱정하지 마세요!

2단계: Node.js 설치

Claude Code를 설치하려면 Node.js가 먼저 필요합니다.

Node.js = Claude Code를 돌리기 위한 엔진. 자동차의 엔진처럼, 한 번 설치하면 신경 쓸 필요 없습니다.
🪟 Windows
1. https://nodejs.org 접속
2. LTS 버전 (초록색 버튼) 다운로드
3. 다운로드된 파일 실행
4. Next → Install → Finish
5. Windows키 → PowerShell 검색 → 실행
6. node --version 입력 → 숫자 나오면 성공!
🍎 Mac
1. https://nodejs.org 접속
2. LTS 버전 (초록색 버튼) 다운로드
3. 다운로드된 .pkg 파일 실행
4. 계속 → 설치 → 비밀번호 → 완료
5. Cmd+Space → 터미널 검색 → 실행
6. node --version 입력 → 숫자 나오면 성공!

3단계: Claude Code 설치

🪟 Windows
1. PowerShell을 엽니다
2. 아래를 타이핑하고 Enter:
npm install -g @anthropic-ai/claude-code
3. 1~2분 기다립니다
4. added ... package 나오면 성공!
🍎 Mac
1. 터미널을 엽니다
2. 아래를 타이핑하고 Enter:
sudo npm install -g @anthropic-ai/claude-code
3. 비밀번호 입력 (입력해도 화면에 안 보이는 게 정상)
4. added ... package 나오면 성공!

설치 확인:


claude --version

버전 숫자가 나오면 성공! ✅

4단계: Claude Code 최초 실행

설치 후 처음 claude를 실행하면 로그인과 권한 설정이 필요합니다.

  • 터미널에서 claude 입력 → Enter
  • 로그인 화면: 브라우저가 열리면서 Claude 계정 로그인을 요청합니다 → 로그인하면 자동 연결
  • 파일 접근 권한: "Allow Claude to access files?" 같은 질문이 나옵니다 → Yes 선택
  • 내 컴퓨터의 파일을 읽고 편집할 수 있도록 허용하는 것입니다
  • Claude Code는 로컬에서만 동작하며, 파일이 외부로 전송되지 않습니다
  • "Welcome to Claude Code!" 메시지가 뜨면 성공!
보안 참고: Claude Code가 접근하는 파일은 현재 작업 폴더 내로 제한되며, 허용 여부를 매번 확인합니다. 민감한 파일에 대해서는 개별적으로 거부할 수 있습니다.

자주 묻는 질문

Q: Node.js가 뭔가요? 위험한 건 아닌가요?

A: Node.js는 프로그램을 실행시키는 엔진입니다. 전 세계에서 가장 많이 사용되는 안전한 도구이며, Claude Code를 돌리기 위해 필요합니다.

Q: npm이 뭔가요?

A: Node.js에 포함된 프로그램 설치 도구입니다. 앱스토어처럼 프로그램을 다운받아 설치해줍니다.

Q: 회사 보안 정책으로 설치가 안 돼요.

A: IT 보안팀에 "npm global 패키지 설치 허용" 요청이 필요할 수 있습니다. 교육 전에 미리 확인해주세요. 안 되면 교육 당일 보조강사가 도와드립니다.

Q: 기존에 쓰던 프로그램에 영향을 주나요?

A: 아닙니다. Node.js와 Claude Code는 독립적으로 설치되며 기존 프로그램에 영향을 주지 않습니다.

Q: 교육이 끝나면 삭제해도 되나요?

A: 네, 삭제 방법:


npm uninstall -g @anthropic-ai/claude-code

Playwriter 설치 (브라우저 자동화)

Playwriter는 현재 사용 중인 Chrome 브라우저(로그인/쿠키 그대로)를 자동 제어하는 도구입니다.

Step 1: Chrome 확장 설치

  • Chrome 웹스토어에서 Playwriter 검색 → 설치
  • 우상단 퍼즐 아이콘(🧩) 클릭 → Playwriter 옆 📌 클릭해 툴바에 고정

Step 2: CLI 전역 설치


npm i -g playwriter

확인:


playwriter --version

Step 3: (선택) 스킬 추가


npx -y skills add remorses/playwriter

사용 시작


playwriter session new

성공 출력:


CDP relay server started successfully
Extension connected
Session 1 created. Use with: playwriter -s 1 -e "..."

아이콘 색상 확인:

  • 🟢 초록 = 연결 성공
  • 🟠 주황 = 연결 중
  • 🔴 빨강 = 에러
  • ⚪ 회색 = session new 먼저 실행

사용 예시


playwriter -s 1 -e "await page.goto('https://example.com')"
playwriter -s 1 -e "await page.screenshot({ path: 'shot.png', fullPage: true })"
playwriter -s 1 -e "console.log(await page.title())"

Dooray MCP 설치

할 것: Claude Code에서 두레이 태스크/위키/드라이브/메신저를 직접 다룰 수 있게 연결합니다.

기본 설치(준비 1)가 완료된 후 진행해주세요. Dooray 실습을 하려면 이 단계가 필요합니다.

Step 1: Dooray API 토큰 발급

  • 브라우저에서 Dooray 접속 (https://nhn.dooray.com)
  • 우측 상단 프로필 아이콘 클릭
  • 설정 클릭
  • 좌측 메뉴에서 API 토큰 클릭
  • 토큰 생성 버튼 클릭
  • 생성된 토큰을 메모장에 복사해두세요

토큰 형식: 조직코드:토큰문자열
예시: ajjt1imxmtj4:tyFuu5obTEGSO3sh5cxgow
이 토큰은 비밀번호와 같습니다. 다른 사람에게 공유하지 마세요!

Step 2: 환경변수 등록

터미널에서 아래를 실행합니다 (토큰을 복사한 값으로 교체):

🪟 Windows
PowerShell에서:
[System.Environment]::SetEnvironmentVariable("DOORAY_API_TOKEN", "여기에_복사한_토큰", "User")

설정 후 PowerShell을 닫고 새로 열어야 반영됩니다.
🍎 Mac
터미널에서:
echo 'export DOORAY_API_TOKEN=여기에_복사한_토큰' >> ~/.zshrc
source ~/.zshrc

Step 3: Dooray MCP 설치

Claude Code에 Dooray MCP를 등록합니다. 터미널에서:


claude mcp add -s user dooray-mcp npx @geonheeye/dooray-mcp@latest
이 한 줄이면 끝! 태스크, 위키, 드라이브, 메신저까지 전부 연결됩니다.

Step 4: 확인

Claude Code를 새로 시작한 후:


내 두레이 태스크 보여줘

내 업무 목록이 나오면 성공! ✅

에러 1: Node.js가 없다 / 버전이 낮다

증상:


'node'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.


Error: Claude Code requires Node.js version 18 or higher

원인: Node.js가 설치되지 않았거나, 18 미만 구버전

해결:

  • https://nodejs.org 접속
  • LTS 버전 (초록색 버튼) 다운로드 → 설치 (모두 Next)
  • 중요: 터미널을 닫고 새로 열기 (안 닫으면 반영 안 됨)
  • 확인:
    • 
   node --version

→ v18.x.x 이상이면 성공

그래도 안 되면: 네이티브 설치로 우회 (맨 위 참고)

에러 2: npm 권한 오류 (EACCES / EPERM)

증상:


EACCES: permission denied, access '/usr/local/lib/node_modules'
EPERM: operation not permitted, unlink '...'
Error: EACCES: permission denied, mkdir '/root/.npm'

원인: npm이 글로벌 폴더에 쓸 권한이 없음

해결 (Windows):

  • 현재 명령 프롬프트 닫기
  • Windows 키 → "cmd" 검색
  • 우클릭 → "관리자 권한으로 실행"
  • 재시도:
    • 
   npm install -g @anthropic-ai/claude-code

해결 (Mac):


sudo npm install -g @anthropic-ai/claude-code

비밀번호 입력 (화면에 안 보이는 게 정상) → Enter

주의: sudo는 이 한 번만 쓰세요. 평소에는 쓰지 않는 게 좋습니다.

더 깔끔한 해결: 네이티브 설치로 우회 (맨 위 참고)

에러 3: 설치 후 "claude를 찾을 수 없습니다"

증상:


Windows: 'claude'은(는) 내부 또는 외부 명령... 이 아닙니다
Mac:     zsh: command not found: claude
         bash: claude: command not found

원인: 설치 경로가 시스템 PATH에 없음

해결 (Windows):

  • 터미널을 완전히 닫고 새로 열기
  • 그래도 안 되면 PowerShell에서:
    • 
   $env:PATH += ";$env:APPDATA\npm"
  • 영구 적용:
  • Windows 키 → "환경 변수" 검색 → "시스템 환경 변수 편집"
  • "환경 변수" 버튼 클릭
  • 사용자 변수 → Path → 편집 → 새로 만들기
  • %APPDATA%\npm 추가 → 확인

해결 (Mac):


echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

Bash 사용자는 ~/.zshrc 대신 ~/.bashrc

에러 4: PowerShell 스크립트 실행 차단 (Windows)

증상:


이 시스템에서 스크립트를 실행할 수 없으므로 ... 를 로드할 수 없습니다
cannot be loaded because running scripts is disabled on this system

원인: Windows PowerShell 보안 정책이 스크립트 실행을 차단

해결:

  • PowerShell을 관리자 권한으로 실행
  • 아래 명령어 입력:
    • 
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  • Y 입력 → Enter
  • 재시도

에러 5: 네트워크/프록시 오류 (회사 방화벽)

증상:


npm ERR! network request failed
npm ERR! code ETIMEDOUT
npm ERR! code UNABLE_TO_GET_ISSUER_CERT_LOCALLY
SSL/TLS secure channel error
unable to get local issuer certificate

원인: 회사 방화벽/프록시가 다운로드를 차단

해결 1 — 프록시 설정 (IT팀에 프록시 주소 문의):


npm config set proxy http://프록시주소:포트
npm config set https-proxy http://프록시주소:포트
npm install -g @anthropic-ai/claude-code

해결 2 — SSL 인증서 검증 비활성화 (임시):


npm config set strict-ssl false
npm install -g @anthropic-ai/claude-code
npm config set strict-ssl true    ← 설치 후 반드시 되돌리기

해결 3 — 회사 인증서 등록:


npm config set cafile /path/to/company-ca-cert.pem

해결 4 — 핸드폰 핫스팟으로 연결 후 설치

가장 빠른 우회 방법. 설치 후에는 회사 Wi-Fi로 돌아와도 됨.

에러 6: 로그인/인증 실패

증상:


OAuth error: Invalid code
403 Forbidden
Not logged in or token expired
API Error: This organization has been disabled

원인: 인증 토큰 만료, 계정 문제, 또는 환경변수 충돌

해결:

  • 로그아웃 후 재로그인:
    • 
   claude /logout
   claude
  • 브라우저에서 https://claude.ai 로그인이 되는지 먼저 확인
  • 브라우저가 자동으로 안 열리면 → 터미널에 표시된 URL을 직접 복사해서 브라우저에 붙여넣기

"organization disabled" 오류:

  • 이전 직장 등에서 설정한 ANTHROPIC_API_KEY 환경변수가 남아있을 수 있음
  • 해결:
    • 
  # Mac/Linux
  unset ANTHROPIC_API_KEY

  # Windows PowerShell
  Remove-Item Env:ANTHROPIC_API_KEY
  • 재시도: claude

에러 7: 한글 깨짐 / 인코딩 오류

증상:


UnicodeEncodeError: 'cp949' codec can't encode character
결과물에 ?????나 깨진 문자가 나옴

원인: Windows의 기본 인코딩(cp949)과 Claude Code(UTF-8) 불일치

해결 (cmd):


chcp 65001

→ 이후 claude 실행

해결 (PowerShell):


[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$env:PYTHONIOENCODING = "utf-8"
chcp 65001

근본 해결: cmd 대신 Windows Terminal 사용 권장

  • Microsoft Store에서 "Windows Terminal" 검색 → 설치
  • 기본 인코딩이 UTF-8이라 한글 문제 없음

에러 12: 중복 설치 충돌

증상:


이전 버전이 실행됨
설치했는데 반영이 안 됨

원인: npm, 네이티브 설치, Homebrew 등 여러 경로로 중복 설치

해결 — 전부 삭제 후 재설치:


# npm 버전 삭제
npm uninstall -g @anthropic-ai/claude-code

# 네이티브 설치 삭제 (Mac/Linux)
rm -f ~/.local/bin/claude

# Homebrew 삭제 (Mac)
brew uninstall --cask claude-code

# Windows
winget uninstall Anthropic.ClaudeCode

그 다음 네이티브 설치로 깨끗하게:


# Windows
irm https://claude.ai/install.ps1 | iex


# Mac
curl -fsSL https://claude.ai/install.sh | bash

설치 후 상태 점검: /doctor

설치는 됐는데 뭔가 이상하면:


claude /doctor

이 명령어가 설치 상태, PATH, 설정 파일, MCP 등을 자동 점검해줍니다.

Claude Code 권한 모드 + 고급 실행 모드

Claude Code 실행 시 나오는 1, 2, 3 선택지의 차이와, 자동화에 유용한 고급 모드를 설명합니다.

권한 모드: 1, 2, 3 선택지

Claude Code가 파일을 읽거나 명령을 실행하기 전에, 이렇게 물어봐요 👇

화면에 뜨는 그대로

번호 화면 문구 의미
1 Yes 이번 한 번만 허용
2 Yes, and don't ask again 이 명령어는 앞으로 묻지 말고 항상 실행
3 No 취소

🤔 그래서 뭘 누르면 되냐면

👉 그냥 1번 (Yes) 누르세요.

매번 확인 뜨는 게 귀찮아지면 2번 — 근데 처음엔 1번 추천!
어떤 명령이 도는지 감을 잡아야, 안전하게 2번을 쓸 수 있어요.

비유로 이해하기

  • 1번 (Yes) = 방문객에게 "이번에만 들어오세요" — 매번 다시 물어봄
  • 2번 (Yes, don't ask again) = "같은 일은 앞으로 묻지 않을게요" — 출입증 발급
  • 3번 (No) = "아뇨, 하지 마세요" — 그냥 취소

어떤 걸 누르면 좋을까요?

상황 권장
교육 중 실습 (처음 며칠) 1번 (Yes) — 뭐가 실행되는지 눈으로 확인하세요
같은 명령이 반복돼서 귀찮을 때 2번 (Yes, don't ask again) — 한 번만 허용해두면 다시 안 물어봐요
파일 삭제/수정 요청 1번 (Yes) — 중요 파일은 매번 확인하는 게 안전
"이거 좀 위험할 것 같은데..." 싶을 때 3번 (No) — 일단 취소하고 다시 물어보면 됩니다
잘못 눌러도 괜찮아요! /permissions 명령어로 언제든 권한 설정을 다시 바꿀 수 있습니다.

Plan Mode (계획 모드)

"계획만 세우고, 실행은 내가 확인한 후에"

기본적으로 Claude Code는 계획과 실행을 동시에 합니다.

Plan Mode를 켜면 계획만 먼저 보여주고, 승인 후에 실행합니다.

켜는 방법:

  • 대화 중 Shift+Tab 두 번 누르기
  • 또는 프롬프트에 "계획만 먼저 세워줘, 실행하지 마" 라고 말하기

사용 예시:


[Plan Mode ON]
> 이 엑셀 데이터를 분석해서 보고서 만들어줘

Claude Code 응답:
📋 계획:
1. 매출_데이터.xlsx 읽기
2. 팀별 매출 합산
3. 전월 대비 증감률 계산
4. 보고서.md 파일 생성

이대로 진행할까요?

언제 쓰나:

  • 복잡한 작업의 방향을 먼저 확인하고 싶을 때
  • "엉뚱한 방향으로 가면 어쩌지?" 걱정될 때
  • 실행 전에 어떤 파일을 건드리는지 미리 보고 싶을 때
비유: 인테리어 업체에 "견적서 먼저 주세요, 바로 공사 들어가지 마세요" 하는 것과 같습니다.

Ralph Mode (자율 실행 모드)

"끝날 때까지 알아서 해줘"

일반 모드에서는 매 단계마다 확인을 받지만, Ralph Mode는 목표를 주면 완료까지 자동으로 진행합니다.

사용 방법:


/ralph 이 폴더의 엑셀 5개를 분석해서 종합 보고서 만들어줘

동작 방식:

  • 목표를 분석해서 계획 수립
  • 각 단계를 자동으로 실행
  • 에러가 나면 자동으로 수정 시도
  • 모든 단계 완료 후 결과 보고

일반 모드 vs Ralph Mode:

매 단계 확인 물어봄 자동 진행
에러 발생 시 멈추고 보고 자동 수정 시도
적합한 작업 한두 단계 작업 여러 단계 복합 작업
비유 수동 기어 자율주행

언제 쓰나:

  • 파일 여러 개를 한꺼번에 처리할 때
  • "이거 하나 끝내면 저거도 해줘" 같은 연쇄 작업
  • 에러가 나도 알아서 해결했으면 좋겠을 때
Ralph Mode는 스킬 설치가 필요합니다 (준비 3: 스킬 설치 참고)

--dangerously-skip-permissions (권한 확인 건너뛰기)

"모든 권한 확인을 생략하고 바로 실행"


claude --dangerously-skip-permissions

이 모드로 실행하면 1, 2, 3 선택이 한 번도 나오지 않습니다. 모든 파일 읽기/쓰기/명령 실행을 자동 승인합니다.

주의사항:

  • 이름에 "dangerously"가 들어간 이유가 있습니다 — 신중하게 사용하세요
  • 실수로 중요한 파일을 덮어쓰거나 삭제할 수 있습니다
  • 개인 실습/자동화 스크립트 실행용으로만 사용을 권장합니다
  • 업무용 중요 폴더에서는 사용하지 마세요

1. claude.ai (웹)

이런 분께 추천

  • AI를 처음 써보는 분
  • 간단한 질문, 문서 검토, 브레인스토밍이 필요한 분
  • 별도 설치 없이 바로 쓰고 싶은 분

할 수 있는 것

  • 채팅으로 질문하고 답변 받기
  • 파일 업로드해서 분석 요청 (PDF, 이미지, 엑셀 등)
  • Research 모드: 웹 검색 + Google Workspace 연동 리서치
  • 음성 대화 (Voice 모드)
  • Projects: 관련 대화와 파일을 묶어서 관리
  • Memory: 내 정보를 기억시켜 매번 설명 안 해도 됨

아쉬운 점

  • 파일을 일일이 업로드해야 함 (내 컴퓨터 직접 접근 불가)
  • 결과물을 복사해서 붙여넣기해야 함
  • 파일을 직접 만들거나 수정해주지 못함
  • 매번 컨텍스트를 새로 설명해야 할 때가 있음

2. Cowork (데스크톱 앱 내 기능)

오늘 교육에서 다루지 않았지만, 비개발자에게 가장 추천하는 기능입니다!

Cowork란?

Claude 데스크톱 앱에 있는 자율 작업 모드입니다. 할 일을 설명하면 Claude가 혼자서 알아서 처리하고, 끝나면 결과물을 보여줍니다.

claude.ai와 뭐가 다른가요?

비교 claude.ai Cowork
작업 방식 내가 물어보면 답변 맡기면 알아서 처리
파일 접근 업로드만 내 컴퓨터 파일 직접 접근
결과물 텍스트 답변 파일 직접 생성 (엑셀, PPT 등)
작업 중 내가 지켜봐야 함 백그라운드에서 진행 (다른 일 가능)
반복 작업 매번 수동 예약 실행 가능

이렇게 씁니다

  • Claude 데스크톱 앱 실행
  • 상단 Cowork 탭 클릭
  • 작업 설명 입력 (예: "Downloads 폴더의 엑셀 파일 10개를 하나로 합쳐줘")
  • Claude가 자율적으로 작업 시작
  • 완료되면 결과 확인

비개발자 활용 예시

엑셀 데이터 정리

"이 폴더에 있는 월별 매출 엑셀 12개를 합쳐서, 분기별 요약 시트 추가하고, 차트도 넣어줘"

보고서 자동 생성

"이 폴더의 일일 업무일지를 읽고, 주간보고 양식에 맞춰 PowerPoint로 만들어줘"

파일 정리

"Downloads 폴더를 파일 종류별로 분류하고, 문서는 Projects 폴더로 옮겨줘"

영수증 처리

"첨부된 영수증 50장에서 날짜, 금액, 가맹점을 추출해서 엑셀로 정리해줘"

주의사항

  • 인터넷 검색은 불가 (내 파일만 작업)
  • 데스크톱 앱이 켜져 있어야 함
  • Pro($20/월) 이상 구독 필요

3. Code 앱 (데스크톱)

오늘 교육에서 쓴 건 사실 이것!

Claude Code 데스크톱 앱의 Code 탭은 시각적 인터페이스로 코드를 편집합니다.

주요 특징

  • Diff 뷰: 파일 변경 전/후를 나란히 비교
  • 실시간 미리보기: 웹앱, HTML 결과를 바로 확인
  • 터미널 내장: Ctrl+\` 으로 터미널 열기
  • 여러 세션 병렬 관리: 여러 프로젝트 동시 작업

4. Code CLI (터미널)

오늘 교육에서 설치한 것!

터미널에서 claude 명령어로 실행하는 방식입니다.

자주 쓰는 명령어


claude                     대화 시작
claude "이거 해줘"          한 줄 명령
claude -c                  이전 대화 이어하기
claude -p "분석해줘"        한 번 실행하고 종료
/exit                      종료

Code 앱 vs Code CLI 차이

구분 Code 앱 Code CLI
인터페이스 그래픽 (시각적) 터미널 (텍스트)
변경 사항 확인 Diff 뷰로 한눈에 텍스트로 확인
미리보기 내장 브라우저 별도 브라우저 필요
리눅스 지원 불가 가능
자동화/스크립트 연동 어려움 쉬움
초보자 친화도 높음 낮음
기능은 동일합니다. 보이는 방식만 다릅니다.

├─ 간단한 질문, 문서 검토, 리서치?

│

비개발자 추천 조합

claude.ai (빠른 질문용) + Cowork (파일 작업용) + Code CLI (자동화용)
일상적인 질문은 claude.ai에서, 파일이 필요한 작업은 Cowork에서,
반복 업무 자동화는 Code CLI에서 처리하면 가장 효율적입니다.

보너스: Git이 뭔가요?

Code를 쓰다 보면 "commit", "branch", "push" 같은 단어가 나옵니다. 이게 다 Git 관련 용어예요.

한 마디로

Git은 파일 변경 이력을 추적하는 시스템입니다.

Git 없이 일하면


보고서_v1.xlsx
보고서_v2_수정.xlsx  
보고서_v2_최종.xlsx
보고서_v2_최종_진짜최종.xlsx   ← 이런 경험 있죠?

Git으로 일하면


보고서.xlsx  ← 파일 하나인데, 모든 버전이 기록되어 있음
             언제든 과거 버전으로 돌아갈 수 있음
             누가 뭘 바꿨는지 다 보임

핵심 용어 4개만

용어 뜻 비유
Repository Git으로 추적하는 폴더 "이 폴더는 관리 중"
Commit 변경 사항 저장 "세이브 포인트"
Branch 본판 건드리지 않고 실험하는 복사본 "시험 삼아 해보기"
Push/Pull 서버에 올리기 / 받아오기 "공유 드라이브에 동기화"

Git vs GitHub

Git GitHub
내 컴퓨터에서 버전 관리 Git 저장소를 올리는 웹사이트
도구 (프로그램) 서비스 (플랫폼)
오프라인 가능 인터넷 필요
Git = 내 책상 서랍, GitHub = 공유 클라우드 드라이브

비개발자가 알아야 할 것

Claude Code를 쓸 때 Git은 뒷단에서 자동으로 돌아갑니다. Claude가 파일을 수정하면 자동으로 commit(저장)하고, 문제가 생기면 되돌릴 수 있게 해줍니다. 직접 Git 명령어를 칠 필요는 없습니다.
