✨ 시작 전에 — 오늘 가장 비싼 도구를 알려드릴게요

⏱ 3분

그건 코딩이 아니라, 사전입니다 📖

같은 Claude Code를 써도 결과가 다른 이유는, "내가 AI에게 뭐라고 부르는지"가 다르기 때문입니다.

모르는 사람 vs 아는 사람 — 같은 일, 다른 결과

영역 모르는 사람의 프롬프트 아는 사람의 프롬프트
데이터 분석 "이 숫자들 중에 평균에서 많이 벗어난 거 찾아줘" "이상치 탐지해줘"
보고서 작성 "결론 먼저, 근거 나중에 쓰는 식으로" "두괄식으로 / BLUF로"
회계/정산 "이 금액 중에 좀 이상한 거 찾아줘" "대사 작업해줘"
계약 검토 "우리가 불리하지 않게 조건 봐줘" "독소조항 체크해줘"
👉 풀어 설명하면 AI가 "많이"의 기준을 멋대로 잡아버립니다.
개념어 하나가 프롬프트 50줄을 이깁니다.

🎬 강사 라이브 시연 (90초)

  1. 매출 데이터 열어두고 모르는 사람 버전 → "평균에서 많이 벗어난 거 찾아줘"
  2. 결과 보여주기 (Claude가 "많이"의 기준을 임의로 잡음)
  3. 같은 데이터에 아는 사람 버전 → "이상치 탐지해줘"
  4. 결과 비교 (IQR/Z-score 자동 적용된 깔끔한 결과)
  5. 한 마디: "달라진 건 AI가 아니에요. 제가 쓴 단어예요."

핵심 메시지: 같은 AI 앞에서 격차를 만드는 건 코딩 실력이 아니라 어휘력이라는 사실. 이 한 줄이 강의 전체를 관통하는 닻이 됩니다.

주의: "프롬프트 엔지니어링"이라는 진부한 표현 X. "사전"이라는 비유로 시작해야 즉시 청중이 집중함.

✨ 시작 전에 — 처음 듣는 단어 몇 개세요?

⏱ 2분

🙋 자가진단 — 손들기 게임

아래 단어 중 의미를 정확히 모르는 게 3개 이상이면 손들어 주세요.

영역 개념어 한 줄 설명
분석이상치 (outlier)평균에서 통계적으로 벗어난 값
보고서두괄식 / BLUF결론 → 근거 순서
마케팅PAS 구조Problem → Agitation → Solution
회계대사 (reconciliation)두 장부 금액을 맞춰보는 작업
인사BEI 질문행동사건면접 (과거 실제 행동 묻기)
협상BATNA협상 결렬 시의 차선책
회의타임박싱어젠다별 시간 상한 설정
UX닐슨 휴리스틱 10사용성 평가 10원칙
계약독소조항일방에 매우 불리한 조항

왜 단어 하나가 결과를 바꿀까요?

  • ① 압축률: "이상치"(3글자) = "평균에서 표준편차 2배 이상 벗어난 값"(20글자+). 짧고 정확한 토큰일수록 AI가 정답으로 직진합니다.
  • ② 학습 정합성: AI는 전문 매뉴얼에서 이 단어들과 그 결과물을 짝지어 학습했어요. 단어 하나로 정답 템플릿이 자동 호출됩니다.
  • ③ 모호성 누적: "많이"를 한 번 임의로 잡으면, 5단계 거치는 동안 원본과 완전히 다른 결과가 됩니다 (compounding error).
"AI는 내 어휘력의 거울입니다. 내가 흐릿하게 부르면, 흐릿한 답이 돌아와요."
오늘 배울 건 Claude Code 사용법이지만, 진짜 배워야 할 건 'AI에게 뭐라고 부르는지'입니다.

운영 흐름 (2분):

  • 표를 한 줄씩 빠르게 읽으며 "이거 처음 듣는다" 손들기
  • 3개 이상 손드신 분이 있을 때 멘트: "오늘 한 단어라도 가져가시면 본전입니다."

3. Claude Code = 내 컴퓨터 안의 비서

⏱ 5분

웹 Claude vs Claude Code

어디서 브라우저 내 컴퓨터 터미널
파일 복사해서 올려야 함 내 폴더를 직접 읽고 수정
결과물 텍스트로 답변 파일을 직접 만들어줌
기억 매번 새로 설명 CLAUDE.md로 기억
연결 AI만 단독 두레이, 메일 등 연결 가능

비유로 이해하기

  • 웹 Claude = 전화 상담사 (매번 전화해서 설명해야 함)
  • Claude Code = 옆자리에 앉은 비서 (내 컴퓨터를 같이 보면서 일해줌)
📋 강사 노트 (클릭하여 펼치기)

"비유하자면요, 웹 Claude는 콜센터에 전화하는 거예요. 매번 처음부터 설명해야 하죠. Claude Code는 옆자리에 앉아서 내 모니터를 같이 보는 비서예요. 내 파일을 직접 열어보고, 결과물도 바로 만들어줍니다."

핵심 포인트: "내 컴퓨터에서 직접 작업한다"는 차이를 명확히.

npm i -g playwriter

확인:


playwriter --version

npx -y skills add remorses/playwriter

playwriter session new

CDP relay server started successfully
Extension connected
Session 1 created. Use with: playwriter -s 1 -e "..."

playwriter -s 1 -e "await page.goto('https://example.com')"
playwriter -s 1 -e "await page.screenshot({ path: 'shot.png', fullPage: true })"

Claude Code 권한 모드 + 고급 실행 모드

Claude Code 실행 시 나오는 1, 2, 3 선택지의 차이와, 자동화에 유용한 고급 모드를 설명합니다.

권한 모드: 1, 2, 3 선택지

Claude Code가 파일을 읽거나 명령을 실행하기 전에, 이렇게 물어봐요 👇

화면에 뜨는 그대로

번호 화면 문구 의미
1 Yes 이번 한 번만 허용
2 Yes, and don't ask again 이 명령어는 앞으로 묻지 말고 항상 실행
3 No 취소

🤔 그래서 뭘 누르면 되냐면

👉 그냥 1번 (Yes) 누르세요.

매번 확인 뜨는 게 귀찮아지면 2번 — 근데 처음엔 1번 추천!
어떤 명령이 도는지 감을 잡아야, 안전하게 2번을 쓸 수 있어요.

비유로 이해하기

  • 1번 (Yes) = 방문객에게 "이번에만 들어오세요" — 매번 다시 물어봄
  • 2번 (Yes, don't ask again) = "같은 일은 앞으로 묻지 않을게요" — 출입증 발급
  • 3번 (No) = "아뇨, 하지 마세요" — 그냥 취소

어떤 걸 누르면 좋을까요?

상황 권장
교육 중 실습 (처음 며칠) 1번 (Yes) — 뭐가 실행되는지 눈으로 확인하세요
같은 명령이 반복돼서 귀찮을 때 2번 (Yes, don't ask again) — 한 번만 허용해두면 다시 안 물어봐요
파일 삭제/수정 요청 1번 (Yes) — 중요 파일은 매번 확인하는 게 안전
"이거 좀 위험할 것 같은데..." 싶을 때 3번 (No) — 일단 취소하고 다시 물어보면 됩니다
잘못 눌러도 괜찮아요! /permissions 명령어로 언제든 권한 설정을 다시 바꿀 수 있습니다.

고급 실행 모드

Plan Mode (계획 모드)

"계획만 세우고, 실행은 내가 확인한 후에"

기본적으로 Claude Code는 계획과 실행을 동시에 합니다.

Plan Mode를 켜면 계획만 먼저 보여주고, 승인 후에 실행합니다.

켜는 방법:

  • 대화 중 Shift+Tab 한 번 누르기 (한 번 = Plan Mode, 두 번 = Bypass Mode이므로 주의)
  • 또는 프롬프트에 "계획만 먼저 세워줘, 실행하지 마" 라고 말하기

사용 예시:


[Plan Mode ON]
> 이 엑셀 데이터를 분석해서 보고서 만들어줘

Claude Code 응답:
📋 계획:
1. 매출_데이터.xlsx 읽기
2. 팀별 매출 합산
3. 전월 대비 증감률 계산
4. 보고서.md 파일 생성

이대로 진행할까요?

언제 쓰나:

  • 복잡한 작업의 방향을 먼저 확인하고 싶을 때
  • "엉뚱한 방향으로 가면 어쩌지?" 걱정될 때
  • 실행 전에 어떤 파일을 건드리는지 미리 보고 싶을 때
비유: 인테리어 업체에 "견적서 먼저 주세요, 바로 공사 들어가지 마세요" 하는 것과 같습니다.

Ralph Mode (자율 실행 모드)

"끝날 때까지 알아서 해줘"

일반 모드에서는 매 단계마다 확인을 받지만, Ralph Mode는 목표를 주면 완료까지 자동으로 진행합니다.

사용 방법:


/ralph 이 폴더의 엑셀 5개를 분석해서 종합 보고서 만들어줘

동작 방식:

  • 목표를 분석해서 계획 수립
  • 각 단계를 자동으로 실행
  • 에러가 나면 자동으로 수정 시도
  • 모든 단계 완료 후 결과 보고

일반 모드 vs Ralph Mode:

항목 일반 모드 Ralph Mode
매 단계 확인 물어봄 자동 진행
에러 발생 시 멈추고 보고 자동 수정 시도
적합한 작업 한두 단계 작업 여러 단계 복합 작업
비유 수동 기어 자율주행

언제 쓰나:

  • 파일 여러 개를 한꺼번에 처리할 때
  • "이거 하나 끝내면 저거도 해줘" 같은 연쇄 작업
  • 에러가 나도 알아서 해결했으면 좋겠을 때
Ralph Mode는 스킬 설치가 필요합니다 (준비 3: 스킬 설치 참고)

--dangerously-skip-permissions (권한 확인 건너뛰기)

"모든 권한 확인을 생략하고 바로 실행"


claude --dangerously-skip-permissions

이 모드로 실행하면 1, 2, 3 선택이 한 번도 나오지 않습니다. 모든 파일 읽기/쓰기/명령 실행을 자동 승인합니다.

주의사항:

  • 이름에 "dangerously"가 들어간 이유가 있습니다 — 신중하게 사용하세요
  • 실수로 중요한 파일을 덮어쓰거나 삭제할 수 있습니다
  • 개인 실습/자동화 스크립트 실행용으로만 사용을 권장합니다
  • 업무용 중요 폴더에서는 사용하지 마세요

언제 쓰나:

  • 자동화 스크립트를 무인 실행할 때 (예: 매일 9시 자동 보고서 생성)
  • 안전한 작업임을 이미 확인했고, 허용 클릭이 번거로울 때

일반 vs Ralph vs Skip Permissions 비교:

항목 일반 모드 Ralph 모드 Skip Permissions
권한 확인 매번 물어봄 매번 물어봄 전혀 안 물어봄
자동 진행 단계마다 확인 자동 진행 자동 진행
에러 자동 수정 X O X
안전도 가장 안전 안전 주의 필요
비유 수동 기어 자율주행 (안전벨트 있음) 자율주행 (안전벨트 없음)
추천 대상 입문자 중급자 자동화 스크립트용
교육 중에는 일반 모드(기본)로 실습합니다. 익숙해지면 Plan Mode → Ralph Mode 순서로 시도해보세요!

모드 조합 활용 팁

목적 추천 조합
처음 배우는 중 일반 모드 + 1번(한 번만 허용)
실습 시간 일반 모드 + 2번(세션 허용)
복잡한 작업 전 확인 Plan Mode → 확인 → 실행
반복 작업 자동화 Ralph Mode
무인 자동 실행 --dangerously-skip-permissions

에이전트 · 오케스트레이션 · 멀티에이전트

AI 자동화의 3단계를 이해하면 Claude Code를 더 강력하게 쓸 수 있습니다.

한눈에 보기


1. 에이전트 (Agent)
   = AI가 스스로 판단해서 행동하는 것
   
   예: "이 엑셀 분석해줘"
   → Claude가 알아서 파일 열고, 분석하고, 결과 저장

2. 오케스트레이션 (Orchestration)
   = 여러 단계를 순서대로 연결하는 것
   
   예: "엑셀 분석 → 보고서 생성 → 두레이 태스크 등록 → 메신저 알림"
   → 4개 작업이 자동으로 순서대로 진행

3. 멀티에이전트 (Multi-Agent)
   = 여러 AI가 역할을 나눠서 동시에 일하는 것
   
   예: Agent A는 데이터 분석, Agent B는 보고서 작성, Agent C는 발송
   → 병렬로 동시에 처리

비유로 이해하기

단계 비유 실제 예시
에이전트 똑똑한 신입사원 1명 "이 엑셀 분석해서 보고서 만들어줘"
오케스트레이션 신입사원에게 업무 프로세스 매뉴얼 제공 "매주 월요일: 데이터 수집 → 분석 → 보고서 → 발송"
멀티에이전트 팀 전체가 역할 분담 "데이터팀이 분석, 기획팀이 보고서, 운영팀이 발송"

Dooray MCP란?



MCP (Model Context Protocol) = Claude Code가 외부 서비스와 연결되는 통로입니다.


Dooray MCP를 설치하면 Claude Code에서 한국어로 부탁하는 것만으로 두레이의 모든 기능을 쓸 수 있습니다.



비유로 이해하기




Dooray MCP 없이:
  두레이 웹 열기 → 프로젝트 찾기 → 태스크 클릭 → 내용 읽기 → 복사 → Claude에 붙여넣기

Dooray MCP 설치 후:
  "두레이에서 이번 주 내 태스크 보여줘" → 끝!




두레이 웹을 한 번도 열지 않고, Claude Code 터미널에서 모든 두레이 작업을 할 수 있습니다.

파일 업/다운로드: 307 리다이렉트 패턴 (중요!)



파일 관련 API는 2단계 요청으로 동작합니다. 이걸 모르면 401 에러가 나요!




① api.dooray.com에 요청 (리다이렉트 따라가지 않기!)
② 307 응답의 Location URL로 Authorization 포함하여 재요청




Python 복사해서 바로 쓸 수 있는 예제




import requests

TOKEN = 'dooray-api 여기에_토큰'
headers = {'Authorization': TOKEN}

# ── 파일 다운로드 ──
url = f'https://api.dooray.com/project/v1/projects/{pid}/posts/{tid}/files/{fid}?media=raw'
r = requests.get(url, headers=headers, allow_redirects=False)   # 307 받기
r2 = requests.get(r.headers['Location'], headers=headers, stream=True)  # 실제 다운로드
with open('다운로드파일.xlsx', 'wb') as f:
    f.write(r2.content)

# ── 파일 업로드 ──
url = f'https://api.dooray.com/project/v1/projects/{pid}/posts/{tid}/files'
r = requests.post(url, headers=headers,
                   files={'file': ('파일명.xlsx', open('파일명.xlsx', 'rb'))},
                   allow_redirects=False)  # 307 받기
r2 = requests.post(r.headers['Location'], headers=headers,
                    files={'file': ('파일명.xlsx', open('파일명.xlsx', 'rb'))})  # 실제 업로드
print(r2.json())  # 업로드 결과




curl 복사해서 바로 쓸 수 있는 예제




# 다운로드 (2단계)
curl -s -D- -o /dev/null \
  'https://api.dooray.com/project/v1/projects/{pid}/posts/{tid}/files/{fid}?media=raw' \
  -H 'Authorization: dooray-api {TOKEN}'
# → Location 헤더의 URL로 재요청

# 업로드 (2단계)
curl -X POST \
  'https://api.dooray.com/project/v1/projects/{pid}/posts/{tid}/files' \
  -H 'Authorization: dooray-api {TOKEN}' \
  -F 'file=@"/path/to/file"' \
  --max-redirs 0
# → Location 헤더의 URL로 재요청 (같은 Authorization 포함!)

주의사항



    

  • allow_redirects=False — 자동 리다이렉트 시 Authorization 헤더가 누락되어 401 발생
    • 

  • ?media=raw — 프로젝트/드라이브 다운로드 시 필수 (없으면 메타정보만 반환)
    • 

  • ?media=meta — 파일 메타정보(id, name, size, mimeType) 조회할 때
    • 

  • Rate Limiting — 429 응답 시 잠시 후 재시도 (헤더: X-RateLimit-Remaining)
    • 

  • 세션 쿠키 불필요 — API 토큰만으로 모든 파일 업/다운로드 동작 (세션 만료 걱정 없음)
    •
